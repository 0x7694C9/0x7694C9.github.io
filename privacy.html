Privacy Policy

Last updated: March 6, 2026

This policy explains what Aurora processes to operate as a Discord bot and how server owners or users can request data-related actions.

Data Aurora may process

  • Discord IDs (user, guild, channel, role) needed to run commands.
  • Moderation metadata, such as case numbers, actions, reasons, and timestamps.
  • Configuration values required for enabled features in a server.
  • Command usage events and runtime logs for reliability and abuse prevention.

How data is used

  • Execute requested bot features.
  • Maintain moderation history and server safety workflows.
  • Diagnose failures, prevent abuse, and improve feature reliability.

Aurora does not sell personal data.

Retention and deletion

Data is retained only as long as needed to provide the feature that created it, or to preserve moderation context for server staff. Server administrators may request cleanup of stored guild data by contacting support.

Third-party services

Aurora depends on Discord and may call optional APIs (for example, weather providers) when users invoke those commands. Those providers receive only the minimum request data required for the command.

Contact

For privacy questions or deletion requests, contact the maintainer via the support server: Support Discord.
